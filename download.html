기능 점검 결과 (v2.6.1)
📸
카메라 연동✅ 정상
navigator.mediaDevices.getUserMedia() 기반 전/후면 전환, 플래시 토글, 촬영 후 배경 적용.
권한 거부 시 단계별 안내 모달 표시. Android WebView / iOS Safari 모두 지원.
🖼️
갤러리 연동✅ 정상
<input type="file" accept="image/*"> — JPEG·PNG·WebP·HEIC·GIF 지원.
Android WebView: label.click() 우선 트리거로 갤러리 오픈 안정화. 같은 파일 재선택 가능.
💾
이미지 저장 / 공유✅ 정상
커스텀 공유 시트 — 갤러리 저장·카카오톡·이메일·Instagram·페이스북·클립보드 복사·PC 다운로드.
OS 시스템 공유창 이중 표시 제거 완료 (navigator.share 직접 호출 제거).
🎬
영상 저장 (WebM / GIF)✅ 정상
MediaRecorder WebM + gif.js animated GIF 저장 지원. 트리밍 크롭 영역 기준 녹화.
drawFrame 6개 레이어(배경·성에·droplet·파티클·낙서·텍스트) 모두 합성 반영.
🎛️
Density 슬라이더 (Rainy·Frosty·Droplets)🔧 수정됨
renderParticles 루프에서 배열 크기 상한(maxP)을 실시간 체크하여 density 감소 시 즉시 반영.
updateDensity에서 initParticles() 재호출로 파티클 수 완전 재초기화.
🌫️
Foggy Density / Background Blur🔧 수정됨
Density: buildWeather() 재빌드로 안개 레이어 불투명도 실시간 반영.
Background Blur: 0~20px 범위 복원. buildWeather 호출 시 bgPhoto에 filter 즉시 적용.
💧
Natural Drops 이미지 (droplet-tex1)🔧 수정됨
핸드폰 프레임 없는 순수 유리창 물방울 이미지로 교체.
SW 캐시 버전 v3으로 업데이트 + ?v=3 쿼리로 캐시 버스팅 적용.
🔄
Droplets → 다른 텍스처 전환 시 배경 소실🔧 수정됨
setTexture() + buildWeather()에서 bgPhoto를 동기로 즉시 복원 (RAF 타이밍 충돌 제거).
🔘
NavBar Reset 버튼🔧 수정됨
drawCanvas가 bottomNav 터치를 가로채는 문제 → onTouchStart/onMouseDown에서 bottomNav 영역 터치 통과 처리.
⚙️
Admin — 광고 포인트·횟수 설정✨ 신규
admin.html 설정 탭에서 하루 최대 광고 횟수, 1단계(15초/+3P), 2단계(30초/+6P) 직접 설정 가능.
localStorage(rn_ad_config) 저장 → 앱 새로고침 즉시 반영.
📱
PWA / 홈 화면 추가✅ 정상
manifest.json: name·short_name·icons(72~512px)·display:standalone·orientation:portrait.
Service Worker v3: Cache-First + 구버전 캐시 자동 삭제.
⚠️
Google Play TWA 등록 주의사항⚠️ 참고
1) icons/icon-192.png, icon-512.png 생성 필수 (icons/generate-icons.html 실행).
2) HTTPS 배포 필수 (카메라·미디어 API는 보안 컨텍스트 요구).
3) assetlinks.json: /.well-known/assetlinks.json 배포 후 TWA 등록.
4) 카메라 권한: AndroidManifest에 CAMERA 퍼미션 명시 필요.
5) 갤러리 권한: Android 13+ READ_MEDIA_IMAGES 또는 미디어 피커 사용.

Google Play 등록 가이드 (TWA)
1
아이콘 생성: icons/generate-icons.html 열고 Generate All Icons 클릭 → 72·96·128·144·152·192·384·512px PNG 생성
2
HTTPS 배포: Netlify / Vercel에 배포. netlify.toml / vercel.json 포함됨
3
assetlinks.json: Digital Asset Links에서 생성 후 /.well-known/assetlinks.json에 배포
4
Bubblewrap CLI: npm i -g @bubblewrap/cli → bubblewrap init --manifest https://[배포URL]/manifest.json
5
APK 빌드: bubblewrap build → app-release-signed.apk 생성 후 Google Play Console 업로드
6
Play Console: 앱 설명·스크린샷(최소 2장)·기능 그래픽(1024×500) 준비 후 내부 테스트 트랙 제출
